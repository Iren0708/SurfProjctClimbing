## Соглашения

- **Платформа:** **нативное мобильное приложение** (iOS + Android) — клиент устанавливается из
  сторов. Это даёт доступ к нативным возможностям: системный push
  ([LOGIC-007](09_Логики/LOGIC-007_Запрос-push-разрешения.md)) и хранение сессионных токенов в
  защищённом хранилище ОС (Keychain / Keystore, [LOGIC-001](09_Логики/LOGIC-001_OTP-авторизация.md)).
- **API:** все запросы REST, спецификации — [../api/](../api/) (домены `auth`, `profile`,
  `slots`, `bookings`, `instructors`). В ТЗ указываются точные `operationId`, метод и путь.
- **Числа не хардкодятся:** потолки форматов, прокатный фонд, цены, лимиты приходят из
  данных слота/зоны и ответов API.
- **Дизайн-требования** зафиксированы в [../3-design-brief/](../3-design-brief/). Сквозные
  правила — [00-foundations.md](../3-design-brief/00-foundations.md). Визуальный язык (палитра,
  шрифты) — зона дизайнера; ТЗ задаёт структуру, поведение и микрокопию.
- **Одна запись = одно место** (FR-6, FR-W1): счётчик мест и гостей на экранах записи **отсутствует**.
