## 1. Назначение и контекст

Верхнеуровневый раздел «Профиль». В MVP:

- Просмотр и редактирование **имени** и **телефона** (FR-1).
- **Выход** из аккаунта.
- **Удаление аккаунта** (с подтверждением; активные брони отменяются на бэкенде).
- Справочные пункты: правила клуба, поддержка, версия приложения.

Не содержит списка записей ([SCR-005](SCR-005-my-bookings.md)) и **не управляет уведомлениями**
(push — [BS-002](BS-002-booking-success.md), foundations §8.1).

## 2. Пользователь и цель

| Параметр | Значение |
|----------|----------|
| **Роль** | Клиент |
| **Контекст** | Эпизодический вход: сверить данные, выйти на общем устройстве |
| **Задача** | «Вошёл под своим номером» / «безопасно выйти» |

## 3. Навигация

- **Вход:** таб «Профиль».
- **«Выйти»** → подтверждение → [SCR-001](SCR-001-registration.md).
- **«Удалить аккаунт»** → подтверждение → SCR-001.
- Смена телефона → шаг OTP (как SCR-001, шаг 2) → возврат в просмотр.

## 4. Контент-инвентарь

| Элемент | Поле | Доступ |
|---------|------|--------|
| Имя | `name` | read + edit |
| Телефон | `phone` | read + edit (с OTP) |
| «Редактировать» | — | режим редактирования |
| «Выйти» | — | с подтверждением |
| «Удалить аккаунт» | — | с подтверждением |
| Справочные пункты | статика | read |

Только данные **текущего** клиента (NFR-9).

## 6. Компоненты и поведение

### Редактирование
- Смена **имени** — без OTP; снек «Профиль обновлён».
- Смена **телефона** — OTP на новый номер; снек «Изменения сохранены».

### Выход
- Диалог подтверждения: «Выйти из аккаунта?» — защита от случайного тапа.
- Снек не показывается; обратная связь — переход на SCR-001.

### Удаление аккаунта
- Явное предупреждение о последствиях (активные брони будут отменены).
- Снек «Аккаунт удалён» на SCR-001.

## 7. Состояния

| Состояние | Поведение |
|-----------|-----------|
| **Просмотр** | Read-only поля |
| **Редактирование** | Поля editable + «Сохранить» |
| **Loading** | При сохранении / OTP |
| **Error** | Снек ошибки |

## 8. Критерии приёмки

**AC-001** — Отображаются имя и телефон текущего клиента.

**AC-002** — «Выйти» требует подтверждения и ведёт на SCR-001.

**AC-003** — Смена телефона требует OTP.

**AC-004** — Чужие данные на экране отсутствуют.
