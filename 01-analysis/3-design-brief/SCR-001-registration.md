## 1. Назначение и контекст

Экран — единственная точка входа в приложение для неавторизованного пользователя. Его задача
(job-to-be-done): **дать клиенту попасть внутрь приложения максимально быстро, без обучения и
без пароля** — подтвердив номер телефона, чтобы тут же перейти к списку слотов и записаться на
тренировку.

Экран реализует принцип **минимального порога входа** (см. [`00-foundations.md`](00-foundations.md), §2, P3)
и обслуживает как **первичную регистрацию**, так и **повторный вход** по номеру телефона. Вход —
**трёхшаговый поток** на единой точке входа (без отдельного выбора «регистрация / вход»):

1. **Шаг 1 — Телефон.** Клиент вводит номер телефона.
2. **Шаг 2 — Код из SMS (OTP).** Клиент вводит код, отправленный на номер; есть таймер
   повторной отправки.
3. **Шаг 3 — Имя (только для нового пользователя).** Если номер ещё не зарегистрирован — клиент
   указывает имя. Известный номер этот шаг **пропускает** и сразу попадает в приложение.

## 2. Пользователь и цель

**Кто:** клиент скалодрома «Вертикаль». Часто — **новый пользователь без опыта работы с приложением**,
который открывает его впервые прямо перед тренировкой.

**Контекст использования** (см. [`00-foundations.md`](00-foundations.md), §1):
- в **зале скалодрома**, нередко с мелом на руках;
- **спешит** перед тренировкой — нет терпения на длинные формы и обучение.

**Цель пользователя:** «Хочу записаться на тренировку — быстро войти и не вспоминать пароль».
**Цель экрана:** провести вход за минимально возможное число действий (номер → код → имя для
новых) и сразу отдать пользователя в [SCR-002 «Список слотов»](SCR-002-slot-list.md).

## 5. Структура и иерархия

Каркас — по [`00-foundations.md`](00-foundations.md), §4.1. Таб-бар в НЗ отсутствует. Поток —
три последовательных шага; на шагах 2 и 3 в хедере есть кнопка «Назад» к предыдущему шагу.
Нижний CTA на каждом шаге — фиксированный, всегда видимый, **не перекрытый клавиатурой**.

**Шаг 1 — Телефон:**

```
┌───────────────────────────────┐
│  Вертикаль                    │  ← хедер: заголовок / приветствие
├───────────────────────────────┤
│                               │
│  Войдите, чтобы записаться     │  ← краткое пояснение
│  на тренировку                 │
│                               │
│  Телефон                      │
│  [ +7 ___ ___-__-__ ]         │  ← поле «Телефон» (числовая клавиатура)
│                               │
│  Без пароля — вход по номеру   │  ← подсказка (caption)
│                               │
├───────────────────────────────┤
│  [     Получить код     ]     │  ← фикс. нижний CTA
└───────────────────────────────┘
```

**Шаг 2 — Код из SMS:**

```
┌───────────────────────────────┐
│  ‹ Назад   Подтверждение       │  ← хедер: назад к шагу 1
├───────────────────────────────┤
│  Мы отправили код на           │
│  +7 999 123-45-67              │  ← номер из шага 1
│                               │
│  Код из SMS                    │
│  [ _ _ _ _ ]                  │  ← поле кода (числовая клавиатура)
│                               │
│  Отправить код повторно (00:30)│  ← таймер → ссылка по истечении
│                               │
├───────────────────────────────┤
│  [      Подтвердить      ]    │  ← фикс. нижний CTA
└───────────────────────────────┘
```

**Шаг 3 — Имя (только для нового пользователя):**

```
┌───────────────────────────────┐
│  ‹ Назад   Как вас зовут?       │  ← хедер
├───────────────────────────────┤
│                               │
│  Имя                          │
│  [____________________]       │  ← поле «Имя» (автофокус)
│                               │
│  Так к вам будут обращаться    │  ← подсказка (caption)
│                               │
├───────────────────────────────┤
│  [      Продолжить      ]     │  ← фикс. нижний CTA → SCR-002
└───────────────────────────────┘
```

> Известный (ранее зарегистрированный) номер шаг 3 **пропускает**: после подтверждения кода
> на шаге 2 сессия устанавливается и сразу открывается [SCR-002](SCR-002-slot-list.md).

## 6. Компоненты и поведение

### Шаг 1 · Поле «Телефон» (`phone`)
- Поле с **числовой / телефонной клавиатурой** и **маской ввода** национального формата.
- Автофокус при открытии экрана.
- По одному и тому же номеру выполняется и регистрация, и повторный вход.

### Шаг 1 · Кнопка «Получить код» (CTA)
- Во всю ширину, тач-зона — по [`00-foundations.md`](00-foundations.md), §3.1.
- **Disabled:** пока телефон не прошёл валидацию.
- **Loading:** после нажатия — индикатор, повторные нажатия блокируются → переход на шаг 2.

### Шаг 2 · Поле «Код из SMS» (OTP)
- Числовая клавиатура, 4–6 знаков. Автофокус при переходе.
- **Таймер повторной отправки:** ссылка неактивна с обратным отсчётом.
- **Кнопка «Подтвердить»:** disabled, пока код не введён полностью.
- **Назад** возвращает на шаг 1 без потери номера.

### Шаг 3 · Поле «Имя» (`name`)
- Только для **нового** пользователя.
- **Кнопка «Продолжить»:** disabled, пока имя пусто → SCR-002.

### Повторный вход (UC-1 A1)
- Известный номер после шага 2 → сразу SCR-002, шаг 3 пропущен.

### Запрос разрешения на push
- На этом экране **не запрашивается** — после первой записи в [BS-002](BS-002-booking-success.md).

## 8. Валидации и микрокопия

| Правило | Текст |
|---------|-------|
| Телефон обязателен | «Введите номер телефона» |
| Телефон неверен | «Похоже, номер введён не полностью» |
| Подсказка | «Без пароля — входим по номеру телефона» |
| Над полем кода | «Мы отправили код на `<номер>`» |
| Неверный код | «Неверный код. Проверьте и введите ещё раз» |
| Таймер повтора | «Отправить код повторно (00:30)» → «Отправить код повторно» |
| Имя обязательно | «Укажите, как к вам обращаться» |
| Имя неверно | «Проверьте имя — кажется, тут лишние символы» |
| Подсказка под именем | «Так к вам будут обращаться» |
| Ошибка входа | «Не удалось войти. Попробуйте ещё раз» |

## 9. Доступность и mobile-first

- Крупные поля и CTA (§3.1, §3.2 foundations).
- Числовая клавиатура для телефона и кода.
- Автофокус на активном поле каждого шага.
- Нижний CTA не перекрывается клавиатурой.
- Screen reader: лейблы у всех полей и кнопок.

## 10. Критерии приёмки дизайна

**AC-001** — Шаг 1: только поле «Телефон» и «Получить код»; пароля нет.

**AC-002** — CTA активируется при валидном телефоне.

**AC-003** — «Получить код» → шаг 2 с таймером.

**AC-004** — Повторный вход: после кода сразу SCR-002, шаг 3 пропущен.

**AC-005** — Новый номер: шаг 3 → SCR-002.

**AC-006** — Невалидный телефон блокирует отправку кода.

**AC-007** — Неверный код не выполняет вход.

**AC-008** — Пустое имя блокирует шаг 3.

**AC-009** — Таймер повторной отправки работает корректно.

**AC-010** — Защита от двойной отправки на CTA.
