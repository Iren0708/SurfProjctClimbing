## 1. Назначение и контекст

Детальный просмотр одной брони и точка входа в **отмену записи** (FR-13). Показывает полные
параметры слота, статус, вариант снаряжения, цену и правило 2 часов.

Для статуса **«Отменена скалодромом»** — причина отмены (FR-16, UC-5).

## 2. Пользователь и цель

- **Цель:** проверить детали записи; при необходимости отменить до старта тренировки.
- **Контекст:** понятное объяснение последствий отмены без штрафов (BR-3).

## 3. Навигация

- **Вход:** SCR-005 → тап по брони; push о напоминании/отмене (опционально deep link).
- **«Отменить»** → [BS-003](BS-003-cancel-confirm.md).
- **Назад** → SCR-005.
- Таб-бар скрыт.

## 6. Компоненты и поведение

### CTA «Отменить запись»
- **Visible + enabled:** `status = active` и `start_at` в будущем (UC-4 E1).
- **Hidden/disabled:** уже отменена (UC-4 E2); тренировка началась; `club_cancelled`.

### Отображение статусов
| status | UI |
|--------|-----|
| `active` | Бейдж «Активна»; CTA отмены |
| `cancelled` | «Отменена»; CTA скрыт |
| `late_cancel` | «Поздняя отмена» + текст из foundations |
| `club_cancelled` | «Отменена скалодромом» + **причина** |
| Прошедшая | «Прошедшая» (производное по времени) |

### После отмены через BS-003
- Экран обновляет статус.
- Снек успеха показывает **SCR-006** (foundations §6.2):
  - ранняя: «Бронь отменена»
  - поздняя: текст поздней отмены из foundations

## 8. Микрокопия

Правило отмены и тексты поздней отмены — **только из foundations §6**, не переписывать.

| Контекст | Текст |
|----------|-------|
| CTA | «Отменить запись» |
| Уже отменена | «Запись уже отменена» |
| После старта | «Тренировка уже началась — отмена недоступна» |
| Скалодром | «Отменена скалодромом» |

## 9. Критерии приёмки

**AC-001** — Для активной предстоящей брони доступна отмена.

**AC-002** — Для `club_cancelled` показана причина, CTA отмены отсутствует.

**AC-003** — После поздней отмены отображается корректный статус и снек из foundations.

**AC-004** — Правило 2 часов видно на экране до отмены.
