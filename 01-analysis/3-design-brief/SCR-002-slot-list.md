## 1. Назначение и контекст

**SCR-002 — главный экран приложения** и стартовая вкладка авторизованной зоны. Показывает каталог
доступных групповых тренировок и служит **точкой входа в сценарий записи**.

По умолчанию — слоты на **ближайшие 7 дней** (R-027); больший период — через фильтр дат (FR-3).
Слоты без свободных мест показываются с пометкой **«Мест нет»**. Фильтр «только со свободными
местами» — в [BS-001](BS-001-filters.md), по умолчанию выключен (FR-4).

При отсутствии слотов — empty state **«Пока нет доступных тренировок»** (FR-3, UC-2 E2).

## 2. Пользователь и цель

**Роль — Клиент.** Контекст — зал скалодрома, спешка, мел на руках (foundations §1).

**Цель:** быстро просмотреть ближайшие тренировки и выбрать подходящую (дата, зона/формат,
инструктор, цена, свободные места).

## 4. Контент-инвентарь

| Элемент | Источник | Назначение в UI |
|---------|----------|------------------|
| Дата и время старта | `slot.start_at` | Ключевое число |
| Зона / формат | `zone_format` (название, тип) | Тип тренировки |
| Инструктор | `instructor_info.name` | Кто ведёт |
| Цена | `slot.price` | Стоимость места |
| Всего / свободно мест | `total_seats` / `free_seats` | «N из M» |
| Статус слота | `status` | `cancelled` — не показывать или пометка |

> Лимиты мест **не хардкодятся** — из данных слота (новичковый ≤ 8, опытный ≤ 16 — на бэкенде).

**Уровень экрана:** заголовок, кнопка «Фильтры» + индикатор активных фильтров.

## 5. Структура и иерархия

```
┌─────────────────────────────────┐
│  Тренировки          [ Фильтры ] │  ← индикатор, если фильтры активны
├─────────────────────────────────┤
│  Сегодня, 5 июля                 │  ← группировка по дням
│  ┌───────────────────────────┐  │
│  │ 18:00 · Болдеринг новички  │  │
│  │ Анна · 3 из 8 мест · 1200 ₽│  │
│  └───────────────────────────┘  │
│  ┌───────────────────────────┐  │
│  │ 20:00 · Трассы опытные     │  │
│  │ Мест нет · 1500 ₽          │  │  ← free_seats = 0
│  └───────────────────────────┘  │
├─────────────────────────────────┤
│ [Тренировки] [Мои записи] [Профиль]│
└─────────────────────────────────┘
```

Сортировка — по `start_at` возрастанию. Pull-to-refresh обновляет список.

## 6. Компоненты и поведение

### Карточка слота
- Тап по всей карточке → SCR-003.
- `free_seats = 0` — визуально приглушена + текст «Мест нет» (не только цветом).
- Цена и свободные места — контрастно (§3.2 foundations).

### Фильтры
- Иконка/кнопка «Фильтры» в хедере → BS-001.
- Индикатор активных фильтров (бейдж/точка) при любом применённом фильтре.

### Пагинация
- Подгрузка следующей страницы при скролле (если API возвращает `meta.total` > загружено).

## 7. Состояния экрана

## 9. Критерии приёмки

**AC-001** — По умолчанию показаны слоты на 7 дней с датой/временем, зоной/форматом, инструктором, местами, ценой.

**AC-002** — При `free_seats = 0` карточка помечена «Мест нет», но остаётся в списке (если фильтр «только свободные» выключен).

**AC-003** — Индикатор активных фильтров виден при применённых фильтрах.

**AC-004** — Empty state при пустом расписании — «Пока нет доступных тренировок».
