## 1. Назначение и контекст

Экран фиксирует запись на **одно место** (FR-6): клиент выбирает вариант снаряжения
(своё / прокатное) и подтверждает бронь. Последний шаг перед [BS-002](BS-002-booking-success.md).

Поток записи: SCR-002 → SCR-003 → **SCR-004** → BS-002 — **≤ 3 экранов** до подтверждения (P2).

> **Одна запись = один клиент = одно место.** Счётчик мест и гостей **отсутствует** (FR-W1).

## 2. Пользователь и цель

- **Цель:** выбрать снаряжение, увидеть итоговую цену, подтвердить запись.
- **Контекст:** зал, спешка — минимум полей и действий.

## 3. Навигация

- **Вход:** SCR-003 → «Записаться» (только при `free_seats > 0`).
- **Успех:** → BS-002.
- **Назад:** → SCR-003 (незавершённая бронь не сохраняется).
- Таб-бар скрыт.

## 4. Контент-инвентарь

### 4.2 Параметры брони

| Поле | Тип | Правило |
|------|-----|---------|
| `equipment` | Ввод | `own` / `rental` — сегментированный переключатель или radio |
| Итоговая цена | Производное | **Сервер** возвращает `price_total` (read-only); клиент не пересчитывает |

**Формула отображения (справочно, до ответа сервера):**
- Своё: `price`
- Прокатное: `price + rental_price` (если прокат — отдельный тариф)

## 6. Компоненты и поведение

### Выбор снаряжения (FR-7, FR-8)
- **«Своё снаряжение»** — занимает место в группе, прокатный фонд не уменьшается.
- **«Прокатное снаряжение»** — дополнительно проверяется `free_rental_equipment > 0`.
- Если прокат недоступен (`free_rental_equipment = 0`) — вариант disabled + подсказка
  «Прокат закончился — выберите своё снаряжение или другой слот» (UC-3 E2).
- Дефолтный выбор — на усмотрение дизайнера (рекомендация: «Своё», если прокат ограничен).

### CTA «Подтвердить запись»
- Loading при запросе; повторные тапы блокируются.
- **Успех** → BS-002 с данными созданной брони.
- **Ошибки (FR-9, FR-10, UC-3 E1–E3):**
  - `slot_full` — «Свободных мест больше нет» + актуальное `free_seats` из `details`.
  - Нехватка проката — предложить «своё» или другой слот.
  - Гонка запросов — тот же паттерн отказа, без овербукинга.
  - Сеть — снек ошибки, экран остаётся.

### Итоговая цена
- Пересчёт превью при смене варианта снаряжения (локально по `price`/`rental_price`).
- После ответа API — показывать `price_total` с сервера.

## 9. Критерии приёмки

**AC-001** — На экране нет выбора количества мест — только вариант снаряжения.

**AC-002** — При выборе проката и `free_rental_equipment = 0` вариант недоступен.

**AC-003** — Цена и напоминание об офлайн-оплате видны до подтверждения.

**AC-004** — Успешное подтверждение открывает BS-002, не дублируя снеком.

**AC-005** — Отказ бэкенда при нехватке мест/проката показывает понятное сообщение.
