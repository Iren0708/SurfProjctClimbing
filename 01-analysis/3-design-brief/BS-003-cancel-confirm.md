## 1. Назначение и контекст

Шторка подтверждения **отмены записи** клиентом. Объясняет последствия в зависимости от времени
до старта: ранняя отмена (≥ 2 ч) освобождает место; поздняя (< 2 ч) — нет, но **без штрафов**
(FR-14, FR-15, BR-3).

## 2. Пользователь и цель

**Цель:** осознанно подтвердить отмену, понимая правило 2 часов.

## 3. Навигация

- **Вход:** SCR-006 → «Отменить запись».
- **Подтверждение** → закрытие шторки → SCR-006 (обновлённый статус) + снек на SCR-006.
- **Отмена / закрытие** → SCR-006 без изменений.

> Закрытие по тапу вне шторки **отключено** — только явная кнопка «Не отменять» (критичное подтверждение).

## 4. Контент-инвентарь

| Элемент | Источник |
|---------|----------|
| Заголовок | «Отменить запись?» |
| Краткая сводка брони | дата, формат |
| Предупреждение о последствиях | **динамическое** — сервер/клиент по времени до `start_at` |
| CTA «Да, отменить» | — |
| CTA «Не отменять» | — |

## 5. Структура

**Вариант A — ранняя отмена (≥ 2 ч до старта):**

```
┌─────────────────────────────────┐
│            ▭▭▭                   │
│  Отменить запись?                 │
│                                  │
│  Ср, 9 июля · 18:00              │
│  Болдеринг                       │
│                                  │
│  Место освободится — другие      │
│  смогут записаться.              │
│                                  │
│  [      Да, отменить      ]      │
│  [      Не отменять       ]      │
└─────────────────────────────────┘
```

**Вариант B — поздняя отмена (< 2 ч):**

```
│  Место останется за вами         │
│  (правило 2 часов). Штраф        │
│  не взимается.                   │
```

Тексты поздней отмены — из [foundations §6](00-foundations.md#6-tone-of-voice-и-общая-микрокопия).

## 6. Компоненты и поведение

### Определение ранней/поздней отмены
- **Источник истины — сервер** (R-021): клиент может показать предупреждение по локальному
  расчёту, итоговый тип отмены — из ответа API.
- Граница «ровно 2 часа» = **ранняя** (`≥ 2 ч`).

### CTA «Да, отменить»
- Loading при запросе.
- **Успех:** закрыть шторку → SCR-006 обновляет статус → снек на SCR-006.
- **Ошибка:** снек на **шторке** (остаётся открытой); `already_cancelled`, `slot_started`.

### CTA «Не отменять»
- Закрывает шторку без действия.

## 7. Состояния

| Состояние | Поведение |
|-----------|-----------|
| **Content (ранняя)** | Предупреждение об освобождении места |
| **Content (поздняя)** | Предупреждение без освобождения, без штрафа |
| **Loading** | CTA loading |
| **Error** | Снек на шторке |

## 8. Критерии приёмки

**AC-001** — Два варианта текста предупреждения в зависимости от времени до старта.

**AC-002** — После успешной ранней отмены — снек «Бронь отменена» на SCR-006.

**AC-003** — После поздней отмены — снек с текстом поздней отмены из foundations (не как ошибка).

**AC-004** — Повторная отмена заблокирована с понятным сообщением.
