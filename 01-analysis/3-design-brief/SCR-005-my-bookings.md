## 1. Назначение и контекст

Список **своих бронирований** с актуальным статусом и параметрами слота (FR-12). Включает
активные, отменённые, поздние отмены и **«Отменена скалодромом»** с причиной (FR-16, US-11).

Корневая вкладка таб-бара «Мои записи».

## 2. Пользователь и цель

**Цель:** контролировать свои записи, видеть статусы, перейти к деталям и отмене.

## 3. Навигация

- **Вход:** таб «Мои записи»; BS-002 → «Мои записи»; push об отмене скалодромом (deep link опционально).
- **Выход:** тап по брони → [SCR-006](SCR-006-booking-details.md).

## 4. Контент-инвентарь

Карточка брони:

| Элемент | Источник |
|---------|----------|
| Дата/время | `slot.start_at` |
| Зона/формат | `zone_format` |
| Инструктор | `instructor_info` |
| Снаряжение | `equipment` (own/rental) |
| Статус | `booking.status` |
| Причина | `cancellation_reason` (для `club_cancelled`) |

### Статусы и бейджи

| status | Бейдж | Примечание |
|--------|-------|------------|
| `active` | «Активна» / без бейджа | Предстоящая |
| `cancelled` | «Отменена» | Ранняя отмена |
| `late_cancel` | «Поздняя отмена» | Место не освобождено |
| `club_cancelled` | «Отменена скалодромом» | + причина |
| *(производное)* | «Прошедшая» | `start_at` в прошлом |

## 5. Структура

```
┌─────────────────────────────────┐
│  Мои записи                      │
├─────────────────────────────────┤
│  Предстоящие                      │
│  ┌───────────────────────────┐  │
│  │ 9 июл · 18:00              │  │
│  │ Болдеринг · Своё           │  │
│  └───────────────────────────┘  │
│                                  │
│  Прошедшие и отменённые          │
│  ┌───────────────────────────┐  │
│  │ [Отменена скалодромом]     │  │
│  │ Профилактика зоны          │  │
│  └───────────────────────────┘  │
├─────────────────────────────────┤
│ [Тренировки] [Мои записи] [Профиль]│
└─────────────────────────────────┘
```

Группировка: **Предстоящие** (active, start_at в будущем) / **Остальные** (прошедшие, отмены).
Pull-to-refresh, пагинация.

## 6. Компоненты и поведение

- Тап по карточке → SCR-006.
- Статусы дублируются текстом/иконкой, не только цветом.
- Пустой список: «У вас пока нет записей» + CTA «Найти тренировку» → SCR-002.

## 7. Состояния

| Состояние | Содержимое |
|-----------|------------|
| **Loading** | Скелетон карточек |
| **Content** | Сгруппированный список |
| **Empty** | «У вас пока нет записей» |
| **Error** | Заглушка + «Обновить» |

## 8. Критерии приёмки

**AC-001** — Отображаются все статусы FR-12, включая «Отменена скалодромом» с причиной.

**AC-002** — Предстоящие и прошедшие/отменённые визуально разделены.

**AC-003** — Только брони текущего клиента (NFR-9).

## 9. Трассировка

| Требование | Покрытие |
|------------|----------|
| FR-12 | Список с параметрами и статусом (§4) |
| FR-16 | Статус «Отменена скалодромом» + причина (§4, §5) |
| US-9, US-11 | Контроль записей, отмена скалодромом (§1, §4) |
| UC-4, UC-5 | Просмотр перед отменой; реакция на отмену клуба (§3) |
